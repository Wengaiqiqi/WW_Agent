# W&W Agent CLI 用户手册

一个跑在你电脑上的 AI 助手，能读写文件、跑命令、上网搜索、看图、接飞书/QQ 当机器人。

---

## 一、第一次使用

需要 **Python 3.11+**。在项目根目录打开 PowerShell：

```powershell
.\install.ps1                 # 安装（自动建 .venv 虚拟环境）
.venv\Scripts\Activate.ps1    # 激活环境
```

## 二、启动

```powershell
python cli.py                   # 启动
```

**首次启动会自动弹出模型向导**，跟着走四步即可（选供应商 → 选模型 → 填 API Key → 填 base URL）。如果没有请使用`/model`命令配置模型
Key 填一次就会保存到本地（文件加了 0600 权限、并自动防止提交到 git），以后启动不用再填。

进入后直接打字跟它对话。例如：

```
帮我把当前目录下的 .txt 文件列出来
搜一下「LangGraph 是什么」并总结三点
```

## 三、`/` 命令

输入框里以 `/` 开头的是命令，不是聊天。

| 命令 | 作用 |
|---|---|
| `/help` | 查看所有命令 |
| `/model` | 重新配置模型（换供应商 / 换模型 / 改 Key） |
| `/status` | 查看当前会话状态（模型、轮次、权限等） |
| `/config` | 查看当前生效配置 |
| `/tools` | 列出助手能用的所有工具能力 |
| `/skills` | 列出已安装的技能 |
| `/agents` | 列出后台的专家子进程 |
| `/instructions` | 列出已加载的项目说明文件 |
| `/permissions` | 查看 / 切换权限（`read-only` 只读 / `workspace-write` 可读写 / `danger-full-access` 全放开） |
| `/gateway` | 配置并启动飞书/QQ 机器人（见下） |
| `/comm` | 管理远程协作对端（`list`/`add`/`use <名>`/`rm <名>`） |
| `/task <内容>` | 把任务交给当前远程对端执行 |
| `/chat <内容>` | 给当前远程对端发消息聊天 |
| `/compact` | 清空对话历史、开新记忆线程 |
| `/clear` | 清屏 |
| `/exit` | 退出 |

> 切权限示例：`/permissions workspace-write`

## 四、用 `/gateway` 接飞书 / QQ 机器人

在 `ww-agent` 里输入 `/gateway`，用方向键操作两级菜单：

1. **选平台**：Feishu / Lark 或 QQ Official Bot（回车进入，Esc 返回）。
2. **选操作**：
   - **Setup credentials** — 一步步填凭据（飞书先选 ws 长连接 / webhook 模式）。回车保留原值，Ctrl+C 取消。
   - **Start gateway** — 在当前会话后台启动机器人（会先问并发数，直接回车即可）。
   - **Stop gateway** — 停掉机器人。
   - **View saved credentials** — 查看已存凭据（密钥打码）。
   - **Clear credentials** — 删除已存凭据。
   - **Back to platform list** — 返回上一级。

菜单底部会实时显示最近日志，方便确认是否连上。**退出 `ww-agent` 会同时关掉所有机器人。**
机器人行为：私聊有问必答；群里只在被 `@` 时回复。

### 飞书/Lark 配置字段

先选连接模式（**ws** 长连接 = 无需公网地址，推荐；**webhook** = 需要公网 URL）。

| 字段 | 必填 | 填写说明 |
|------|------|----------|
| `app_id` | ✅ | 飞书开放平台 App ID（`cli_xxxx`） |
| `app_secret` | ✅ | App Secret |
| `domain` | | `open.feishu.cn`（国内，默认）或 `open.larksuite.com`（海外） |
| `allowed_users` | | 逗号分隔的授权 open_id；留空则无人可用 `/chat` `/task` |

Webhook 模式额外字段：`verify_token`（必填）、`encrypt_key`（可选）、`reply_in_thread`（`y`/`n`）、`host`（默认 `0.0.0.0`）、`port`（默认 `8765`）。

### QQ 官方机器人配置字段

| 字段 | 必填 | 填写说明 |
|------|------|----------|
| `app_id` | ✅ | QQ 开放平台 Bot AppID |
| `client_secret` | ✅ | Bot Client Secret |
| `intents` | | **留空即可**，默认接收私聊 + 群聊 @ + 频道 @ 消息 |
| `sandbox` | | `n`（正式环境，默认）；除非在用腾讯沙箱测试频道才填 `y` |
| `allowed_users` | | 逗号分隔的授权 openid；留空则无人可用 `/chat` `/task` |

## 五、用 A2A 跟远程 Agent 协作（`/comm` · `/task` · `/chat`）

A2A（Agent-to-Agent）让你的本机助手通过网络**调用另一台机器上的 Agent**——把任务委派过去，或跟它多轮聊天。通信用 HMAC 签名鉴权，传输走 HTTPS（对端用 Caddy 终止 TLS）。

整体分两个角色：

- **本机 = 调用方**：用 `/comm` 注册并选中对端，再用 `/task` / `/chat` 跟它交互。
- **对端 = 服务方**：在远程机器上跑一个"桥"进程，对外暴露 A2A 服务（搭建见下文「对端怎么搭」）。

### 1. 注册一个远程对端

```
/comm add
```

会逐项提示填写（Ctrl+C 取消）：

| 字段 | 说明 |
|---|---|
| `peer_id` | 对端的唯一标识，例如 `hermes-server` |
| `url` | 对端地址，例如 `https://8.163.112.21:8443`（优先 https） |
| `display_name` | 显示名，留空则同 `peer_id` |
| `Self-signed certificate?` | 对端用自签证书才选 `y`，然后填证书的 SHA-256 指纹 |
| `HMAC secret` | 和对端约定的共享密钥（输入时隐藏） |

注册成功后会自动设为「当前对端」，并尝试拉取对端的 agent card 确认连通。

> **重要：HMAC 密钥不落盘。** 它只写进当前进程的环境变量（出于安全），注册成功时会提示一条 `export COMM_PEER_<名>_HMAC=<值>`。**重启后想免重填，就把这条加到你的 shell profile**（或启动前先 export），否则下次启动该对端会因密钥缺失而连不上。

### 2. `/comm` 子命令

| 命令 | 作用 |
|---|---|
| `/comm list` | 列出已注册对端（★ 标记当前选中的） |
| `/comm add` | 交互式注册新对端 |
| `/comm use <peer_id>` | 切换当前对端（**这个选择会持久化，重启后自动恢复**） |
| `/comm rm <peer_id>` | 删除对端 |

### 3. 委派任务 vs. 聊天

选中一个对端后：

| 命令 | 用途 | 特点 |
|---|---|---|
| `/task <内容>` | 把一个一次性任务**委派**给对端执行 | 对端边做边返回进度，最后给结果 |
| `/chat <内容>` | 跟对端**多轮对话** | 按会话保持上下文，可连续追问 |

示例：

```
/comm use hermes-server
/task 帮我把 logs 目录下今天的报错汇总成一份清单
/chat 你好，介绍一下你能做什么
```

## 六、启动 Web 网页版

想用浏览器界面，新开一个 PowerShell（已激活环境）：

```powershell
python -m web                 # 启动，默认地址 127.0.0.1:8080（仅本机可访问）
```
或者使用start_web.bat脚本
然后浏览器打开 **http://127.0.0.1:8080**，注册账号即可开始聊天。
